# i-route

## 프로젝트 목표 : 
본 프로젝트는 아이의 안전한 학원 통학 환경을 제공하는 통합 플랫폼 ‘아이루트’를 개발하는 것을 목표로 한다.
GPS 기반 위치 추적 기술을 활용하여 학부모가 자녀의 이동 경로를 실시간으로 확인할 수 있도록 하고, AI 분석 기능을 통해 학습 데이터를 기반으로 개인 맞춤형 학습 로드맵 및 학업 방향성을 제시한다.

이를통해
- 학부모의 안전 불안 해소
- 학생의 학습 효율 향상
- 학원과 학부모 간의 커뮤니케이션 개선
을 동시에 달성하는 것을 목표로 한다.

## 프로젝트 기능
본 프로젝트에는 5가지의 핵심 기능으로 구성된다.
① GPS기반의 실시간 위치 추적 서비스
- 기사의 스마트폰 GPS를 활용하여 현재 위치를 실시간으로 수집 및 표시
- 지도 API를 활용하여 이동 경로 시각화 (실시간 트래킹)
- 학부모는 앱을 통해 자녀의 현재 위치, 이동 경로, 학원 도착 여부를 확인 가능
- 특정 구간 (집 -> 학원) 이탈 시 이상 이동 감지 및 알림 제공
② AI 기반 학습 로드맵 추천 시스템
- 학생의 성적, 학습 시간, 문제 풀이 데이터 등을 기반으로 분석
- 머신러닝 또는 규칙 기반 알고리즘을 활용하여 개인 맞춤형 학습 계획을 생성
- 주요 기능 : 과목별 취약 영역 분석, 학습 우선순위 추천, 주간/월간 학습 계획 자동 생성
- 학부모에게 학습 성장 추이 리포트 제공
③ 카카오톡 소셜 로그인 기반 인증 시스템
- 카카오 OAtuh를 활용한 간편 회원가입 및 로그인 기능 구현
- 사용자 유형 구분 : (학부모, 기사, 학원 관리자, 학생)
- 로그인 과정 간소화를 통해 사용자 접근성 향상
④ 학원 커뮤니케이션 플랫폼 기능
- 게시판 기능을 통해 학원 공지사항 확인 및 수업 일정 및 변경사항 확인
- 학부모 – 학원간 1:1 메시지 기능 제공
- 알림 기능을 통해 중요한 공지사항을 실시간 전달
⑤ 안전 알림 및 통학 관리 시스템
- 자녀의 통학 상태를 감지하여 출발/이동 중/ 도착 상태를 구분
- 주요 기능 : 학원 도학 시 자동 알림 전송, 지정 시간 미도착 시 경고 알림 제공, 위험 지역(설정 가능) 접근 시 알림
- 보호자에게 푸시 알림 기반 실시간 상황 전달

## 접근 방법 및 도구:
1) 시스템 아키텍처 설계
- 클라이언트-서버 구조 기반 설계
- - Frontend : 모바일 앱（안드로이드)
- - Backend: RESTful API 서버
- 위치 데이터 및 학습 데이터를 서버에서 통합 관리

2) 위치 기반 서비스 구현
- Google Maps API 또는 Kakao Map API활용
- GPS 데이터를 주기적으로 수집하여 서버에 전송
- WebSocket 또는 Polling 방식으로 실시간 위치 갱신
- 위치 정확도 향상을 위한 보정 로직 적용

3) AI 학습 분석 시스템 구축
- 초기 단계 : 규칙 기반 추천 시스템
- 확장 단계 : 머신러닝 모델 적용
- - 입력데이터 : 성적, 학습 시간, 문제 풀이 기록
- - 출력 데이터 : 학습 계획 및 추천
- Python 기반 분석 모듈을 서버와 연동

4) 인증 및 보안 설계
- 카카오 OAuth 인증 적용
- JWT 기반 사용자 인증 처리
- 개인정보 보호를 위한 암호화 및 접근 제어 구현

5) 데이터베이스 설계
- MySQL 또는 PostgreSQL 사용
- 주요 테이블 구성
- - 사용자(User)
- - 학생 (Student)
- - 위치 기록 (Location History)
- - 학습 데이터(Learning Data)
- - 공지사항 (Notion)
- - 메시지(Mesaage)
